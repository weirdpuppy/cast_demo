    So... What's happenning here?

    This is an example of a basic cast sender that sends messages to the receiver by using the sendGameMessage(...) function provided by chrome.cast.games.GameManagerClient in the Game Manager APIs.

    The sendGameMessage(...) function can be used to send customized messages to the receiver. For example, in your own game, you may want to make your character "jump". You can do this by sendGameMessage({action : 'jump'}).

    In this game we initialized our player with: sendPlayerAvailableRequest(...) instead of sendPlayerAvailableRequestWithPlayerId(...). When a custom playerId is not provided, the receiver will automatically assign a unique playerId to our sender. This means we can use the function sendGameMessage(...) instead of sendGameMessageWithPlayerId(...)just fine.

    In a game that supports multiple players on a single sender device, you may need to send messages to the receiver with regard to specific players. To see how you can create a game with custom playerIds and send player specific messages, check out the sample app StarCast!

    By opening the chrome dev console (You can do so by pressing the F12 key on a PC and Cmd + Opt + I on a Mac.) you can take a look some of the other available commands provided by the Game Manager API.
